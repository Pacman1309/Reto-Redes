Como seguirla

La guia esta completa dentro de este HTML: puedes navegar por paso, copiar bloques de CLI y marcar avance sin abrir el archivo de texto.

Convenciones

  • Cuando diga "abrir CLI", hagan clic en el dispositivo, pestana CLI, presionen Enter si lo pide y peguen el bloque completo.
  • Al final de cada bloque se incluye "write memory" para guardar.
  • Si Packet Tracer pregunta "Continue with configuration dialog?", escriban no.
  • Si un modelo Switch-PT viejo marca error en un comando de STP o trunk avanzado, ignoren solo ese comando y sigan con el resto.
  • Para VTP usen siempre el mismo dominio y password en todos los switches: dominio OMI-REDES, password OMI2026, version 2.
  • No apliquen rangos de puertos sobre uplinks/trunks. Los uplinks son GigabitEthernet; los usuarios/APs normalmente estan en FastEthernet.

Paso 3

Configurar VTP Client En Todos Los Switches Restantes

Dispositivos

SW-1ERA.1-43 SW-1ERA.1-40 SW-2ERA.1-44 SW-2ERA.1-42 SW-3ERA.1-44 SW-3ERA.1-42 SW-1ERC.1-44 SW-1ERC.1-42 SW-1ERC.1-40 SW-4ERC.1-44 SW-4ERC.1-42 SW-1ERB.1-44 SW-1ERB.1-42 SW-2ERB.1-44 SW-2ERB.1-42 SW-2ERB.1-40 SW-2ERBA.1-44 SW-2ERBA.1-42 SW-2ERBL.1-44 SW-2ERBL.1-42

Accion

En cada switch de la lista, abrir CLI y pegar el mismo bloque. Despues cambien el hostname en la primera linea por el nombre real del switch.

enable
configure terminal
hostname NOMBRE_DEL_SWITCH
vtp domain OMI-REDES
vtp mode client
vtp version 2
vtp password OMI2026
end
write memory

Ejemplo

Si estan en SW-2ERA.1-42, cambien:

hostname NOMBRE_DEL_SWITCH

por:

hostname SW-2ERA.1-42

Verificacion en cada switch cliente

show vtp status

Resultado esperado

  • VTP domain name: OMI-REDES.
  • VTP operating mode: Client.
  • La version debe ser 2.
  • Las VLAN 10-80 apareceran despues de que el trunk hacia el servidor VTP quede activo en el Paso 4.

Nota

Si un Switch-PT viejo no acepta "vtp password" o "vtp version 2", apliquen por lo menos "vtp domain OMI-REDES" y "vtp mode client". Si VTP no propaga en ese modelo, usen el bloque manual de VLANs del Paso 2 como respaldo solo en ese switch.

Paso 5

Configurar Puertos De Acceso Por Area

Importante

El XML muestra muchos equipos llegando por TO y patch panels. En Packet Tracer, identifiquen visualmente el puerto exacto donde termina cada TO. Si varios equipos aparecen conectados al mismo puerto por error de patching, repatchen para que cada dispositivo tenga un puerto de switch propio.

5.1 Puertos para LWAP / AP ligeros - VLAN 70 GestionTI

Los AP ligeros deben quedar en VLAN 70 para descubrir el WLC por CAPWAP.

Mapa detectado

  • CIT-PBAP-01, CIT-PBAP-02 -> SW-1ERA.1-40
  • CIT-02AP-01 -> SW-2ERA.1-42
  • CIT-03AP-01, CIT-03AP-02, CIT-03AP-03 -> SW-3ERA.1-42
  • HUM-PBAP-02 y Light Weight Access Point6 -> SW-1ERB.1-42
  • HUM-02AP-05 -> SW-2ERB.1-40
  • HUM-02AP-01, HUM-02AP-02, HUM-02AP-03, HUM-02AP-04 -> SW-2ERBA.1-42
  • HUM-PBAP-03 -> SW-2ERBL.1-42
  • ING-04AP-01 -> SW-4ERC.1-42
  • ING-PBAP-01 -> SW-1ERC.1-40

En cada switch donde haya APs, seleccionen SOLO los puertos fisicos de esos APs y peguen este bloque cambiando el rango si es necesario:

enable
configure terminal
interface range FastEthernet0/1 - 24
 description LWAP_Gestion_CAPWAP
 switchport mode access
 switchport access vlan 70
 spanning-tree portfast
 no shutdown
end
write memory

Si ese switch tambien tiene PCs cableadas, NO usen todo Fa0/1-24. Usen solo los puertos donde realmente llegan los APs, por ejemplo:

interface range FastEthernet0/3 - 6
